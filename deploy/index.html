acme-api — xocode
The orchestration layer

Uses the CLIs you already trust.

xocode doesn't replace your tools — it drives them. Planning runs in Claude Code; building runs in cursor-agent. Your accounts, your models, your history.

Plans the work
Claude Code
Opus 4.8

Runs read-only in plan mode at high effort. Reads your repo, reasons about the change, and streams an implementation plan — it never writes a line until you approve.

Builds the work
Cursor
Composer 2.5

Implements the approved plan in an isolated git worktree, editing files and running commands. Fast, autonomous, and fenced off from your working branch.

How it works

One command. Three moves you already make.

01 · Plan

Opus 4.8 writes the plan

Claude Code runs in read-only plan mode with Opus 4.8 at high effort and streams an implementation plan, saved to .xocode/plans/ in your project.

02 · Review

You read it and approve

Read the plan in the terminal, or open it in $EDITOR and change anything you like. Nothing gets built until you approve.

03 · Build

Composer 2.5 ships it

cursor-agent runs Composer 2.5 to implement the approved plan in an isolated worktree — then prints the exact commands to review, merge, or discard.

Isolated by default

Your branch stays clean.

Every build runs in a dedicated git worktree at ~/.cursor/worktrees/<repo>/<name>. Your working branch is never touched.

  • Review the result before it goes anywhere
  • Merge it when it's right
  • Throw it away with a clean tree if it's not
# builds land here — never on your branch
~/.cursor/worktrees/acme-api/rate-limit

git switch rate-limit        # review it
git merge  rate-limit        # keep it
git worktree remove <path>   # or discard
Commands

Four commands. That's the surface.

xocodeLaunch the interactive TUI — the main way to use it.
xocode doctorCheck prerequisites. --json for CI (non-zero on fail).
xocode upgradeUpdate to the latest release. --check to only report.
xocode versionPrint the version.
Requirements

What you need.

macOS or Linux
arm64 or amd64. A single static binary — no runtime to manage.
Your own accounts
A Claude account and a Cursor account. Both run on your keys.
Guided setup
xocode installs the Claude Code and Cursor CLIs and walks you through login.
